== ラボ概要

このセクションでは、このラボで操作する automation controller UI の主要コンポーネントを紹介します。

=== Ansible Automation Platform へようこそ

次の章では、Web UI ダッシュボードを確認し、**Inventory**、**Projects**、**Job Templates** の各セクションを操作します。

=== Automation Platform の概要

**Overview** 画面はメインダッシュボードです。hosts、projects、inventories の概要が表示され、各数値は対応オブジェクトへのリンクになっています。リソース数の下には Job Activity グラフと、最近の Jobs、Projects、Inventories のカード形式サマリーが表示されます。

image::Oct-16-2024_at_02.01.47-image.png[Automation controller overview dashboard, opts="border"]

=== Automation Controller Jobs ビュー

左側ナビゲーションバーの *Jobs* をクリックすると **Jobs** ビューを開けます。このビューには、プロジェクト同期、SCM 更新、playbook 実行など、これまでに実行されたすべてのジョブが表示されます。

image::ug-dashboard-jobs-view.png[Jobs view in automation controller, opts="border"]

=== Schedules ビュー

左側ナビゲーションメニューの **Automation Execution** → **Schedules** をクリックすると **Schedules** ビューを開けます。このビューには、定期実行するように設定されたスケジュール済みジョブが表示されます。ジョブ起動、インベントリーソース同期、バージョン管理からのプロジェクト内容取り込みに利用できます。

image::Oct-16-2024_at_02.12.32-image.png[Schedules view in automation controller, opts="border"]

新しい統合 UI には複数のセクションがありますが、このラボでは **Automation Execution (Automation controller)** ビューに注目します。

=== Task 1: Overview を確認する

最初のタスクでは **Overview** ダッシュボードを確認します。初期状態では最近のアクティビティが少ないため情報は少なめですが、ラボ完了後に再度確認すると違いが分かります。

次のセクションがあります。

* **Resource counts:** *hosts*、*projects*、*inventories* のサマリー
* **Job activity:** 過去1か月のジョブ実行を示すグラフ
* **Jobs:** 最近実行されたジョブの一覧
* **Projects:** 最近更新されたプロジェクトの一覧
* **Inventories:** 最近更新されたインベントリーの一覧

Ansible Automation Platform 2.5 の新機能として、クイックスタートガイドとインタラクティブな how-to セクションが追加されており、インストール直後の立ち上がりに便利です。

image:Oct-16-2024_at_02.29.14-image.png[Quick start guides on the dashboard, opts="border"]

各タイトルや "View all" リンクをクリックして、対応セクションを自由に確認してください。

=== Task 2: Inventories セクションを確認する

**inventory** は playbook の実行対象となる hosts の集合です。コマンドラインで使う inventory file と同じ概念ですが、追加機能と再利用性があります。

. **Inventories ページへ移動する。**
+
*Overview* ページの **Inventories** リンク、または左側ナビゲーションメニューの **Infrastructure** → **Inventories** から移動します。

. **Demo Inventory を確認する。**
+
事前に用意されている **Demo Inventory** をクリックし、利用可能なフィールドに慣れるために各タブを確認します。後続で独自の inventory を作成します。

=== Task 3: Projects セクションを確認する

**project** は automation controller 内での Ansible Playbook の論理的なグループです。これらの playbook は通常、Git のようなソースコード管理システムに配置されています。

. **Projects ページへ移動する。**
+
左側ナビゲーションメニューで **Automation Execution** → **Projects** に移動します。

. **Demo Project を確認する。**
+
事前に用意されている **Demo Project** をクリックし、利用可能な各フィールドとタブを確認します。
