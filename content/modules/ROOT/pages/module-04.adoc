== ラボ概要

**Project** は Ansible Playbook の論理的なコレクションです。playbook は、Git、Subversion、Red Hat Insights など、automation controller が対応するソースコード管理（SCM）システムで管理できます。

image::projects-list-all.png[Projects list view in automation controller, opts="border"]

---

== ラボガイド: ハンズオンタスク

automation controller の **Project** は、ディレクトリから 1 つ以上の Ansible Playbook（関連する roles やコンテンツを含む）を取り込む仕組みです。automation controller の Web UI で playbook の作成や編集はできませんが、Git リポジトリーのようなソースから Projects を通じて取り込みます。

このチャレンジでは、簡単な Apache playbook を含む GitHub リポジトリーを取り込む最初の Project を作成します。

[IMPORTANT]
====
automation controller の認証情報:

* *Username:* `admin`
* *Password:* `ansible123!`
====

=== Task 1: Project を作成する

最初に、公開 Git リポジトリーを参照する新しい project を作成します。

. **Projects ページへ移動する。**
+
左側ナビゲーションメニューで **Automation Execution** → **Projects** に移動します。

. **新しい project を作成する。**
+
青色の **Create project** ボタンを **クリック** します。

. **project の詳細を入力する。**
+
フォームに以下を入力します。
+
* **Name:** `Apache playbooks`
* **Organization:** `Default`
* **Source Control Type:** `Git`
* **Source Control URL:**
+
[source,text]
----
https://github.com/ansible-tmm/instruqt-wyfp.git
----

. **project を保存する。**
+
他の項目はそのままにして、青色の **Create project** ボタンを **クリック** します。

=== Task 2: Project 同期を確認する

project を作成すると、automation controller は自動的に同期処理を開始し、Git リポジトリーから playbook を取得します。

. **project ステータスを確認する。**
+
保存後、新しい project の *Details* タブに移動します。`Apache playbooks` project の *Status* が緑色で **Successful** と表示されることを **確認** します。

. **同期出力を確認する。**
+
**Successful** のステータステキストを **クリック** して同期ジョブの出力を確認します。Git clone 処理の詳細が表示されます。
