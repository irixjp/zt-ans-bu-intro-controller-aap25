ラボガイド: 複数の Job Template を作成する

プロジェクト内の異なる playbook を実行するために、2つ目と3つ目の Job Template を作成するガイド。

ラボ概要

automation controller では、job templates は ansible-playbook コマンドと、CLI 実行時に使える各種フラグを視覚化したものです。Ansible Playbook を実行する主要な仕組みです。

ラボガイド: ハンズオンタスク

これまでに学んだ内容を使って、2 つ目と 3 つ目の Job Template を作成します。

  • 2 つ目の Job Template Extended services では、extended_services.yml playbook を使って node3 に PostgreSQL データベースをデプロイします。

  • 3 つ目の Job Template Set motd では、motd_facts.yml playbook を使って node1 に message of the day を設定します。

この演習では、同じ Project 内の異なる playbook を複数 Job Template で使い分ける方法を示します。

automation controller へのログイン認証情報:

  • Username: admin

  • Password: ansible123!

Task 1: "Extended Services" Job Template を作成する

最初に、データベースをデプロイする job template を作成します。

  1. Templates ページへ移動する。

    左側ナビゲーションメニューで Automation Execution → Templates に移動します。

  2. 新しい job template を作成する。

    青色の Create template ボタンを クリック し、Create job template を選択します。

  3. job template の詳細を入力する。

    フォームに以下を入力します。

    • Name: Extended services

    • Job Type: Run

    • Inventory: Lab-Inventory

    • Project: Additional playbooks

    • Playbook: extended_services.yml

    • Credentials: lab-credentials を選択

    • Privilege Escalation: become を有効にするため チェック

  4. job template を保存する。

    他の項目はそのままにして、青色の Create job template ボタンを クリック します。

Credentials フィールドは UI 上で必須ではありませんが、正しい machine credentials を選択しないとジョブは失敗します。

Task 2: "Extended Services" Job Template を起動する

次に template を起動し、想定される失敗をトラブルシュートします。

  1. template を起動する。

    Templates ページに移動し、Extended services job template の横にある Launch アイコン（🚀）を クリック します。

  2. 失敗を確認して解消する。

    ジョブ実行は 失敗 します。出力メッセージを読み、理由を確認してください。playbook は [database] group 内の node3 という host を対象にしていますが、Lab-Inventory にどちらも存在しません。

  3. inventory を更新する。

    inventory チャレンジで学んだ内容を使って Lab-Inventory に戻り、次を実施します。

    • node3 という新しい host を 作成

    • database という新しい group を 作成

    • database group に node3 host を 追加

  4. ジョブを再起動する。

    Extended services job template を再度 Launch し、今度は成功することを確認します。

Task 3: "Set motd" Job Template を作成する

次に、学んだ内容を使って 3 つ目の job template を作成します。

  1. Templates ページへ移動する。

    Automation Execution → Templates に移動します。

  2. 新しい job template を作成する。

    Create template ボタンを クリック し、Create job template を選択します。

  3. job template の詳細を入力する。

    フォームに以下を入力します。

    • Name: Set motd

    • Inventory: Lab-Inventory

    • Project: Additional playbooks

    • Playbook: motd_facts.yml

    • Credentials: lab-credentials を選択

    • Privilege Escalation: このボックスを チェック

  4. job template を保存する。

    青色の Create job template ボタンを クリック します。

Task 4: "Set motd" Job Template を起動する

最後に、新しい template を起動して message of the day を設定します。

  1. template を起動する。

    Templates 一覧で、Set motd job template の横にある Launch アイコン（🚀）を クリック します。

  2. 実行成功を確認する。

    ジョブが Successful ステータスで完了すれば成功です。

次のステップ

次のチャレンジに進むには、下の Check ボタンを押してください。
